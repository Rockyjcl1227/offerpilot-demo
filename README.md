# OfferPilot

贾长乐做的，面向大学生求职季的 `求职申请管理看板` 交互式 MVP，。

## 项目简介

OfferPilot 不是一个简单的岗位记录表，而是一个帮助学生管理完整求职流程的工作台。

它要解决的核心问题是：

- 同时申请多个岗位时，信息分散、进度混乱
- 截止日期、测评、面试安排容易遗漏
- 简历、作品集、成绩单等材料准备状态不清楚
- 用户知道自己“很忙”，但不知道今天最该做什么

因此，这个产品的核心目标不是“记录申请”，而是把求职过程变成一条 `可视化、可推进、可分析` 的链路。

## 本次作品的设计方向

本项目围绕两个角色展开：

- `用户端`
  面向求职中的大学生，强调当前状态感知、下一步动作承接、材料补齐和流程推进。

- `管理端`
  面向平台产品和运营视角，强调平台规模、业务指标、过程转化、用户链路和埋点事件分析。

整体设计上，优先保证：

- 页面可交互
- 链路完整
- 叙事清楚
- 适合以链接形式提交和演示

## 当前版本包含什么

### 用户端

- `今日总览`
  通过当前申请数、紧急节点、面试中岗位、待补材料等信息，帮助用户快速判断自己目前的状态。

- `申请看板`
  以看板形式管理岗位阶段，支持：
  - 感兴趣
  - 已投递
  - 测评 / 笔试
  - 面试中
  - 已 Offer

- `日历提醒`
  展示未来七天的重要节点，并提供提醒和跟进建议。

- `个人中心`
  原“材料中心”已调整为“个人中心”，包含：
  - 通用求职材料管理
  - 岗位缺失材料分析
  - AI 简历与材料工作台
  - AI 输入框和发送按钮的交互占位

- `岗位详情抽屉`
  查看某个岗位的完整信息，包括节点、材料、流程记录和下一步建议。

- `新增申请`
  支持新增岗位申请，模拟真实使用场景。

### 管理端

- `平台核心指标`
  包括平台日活、申请活跃率、提醒触达率、状态推进率等假设业务数据。

- `业务概览`
  更偏平台视角，例如活跃申请总量、材料阻塞岗位占比、面试阶段岗位占比。

- `过程指标与转化率`
  例如：
  - 首页 -> 查看岗位详情
  - 查看详情 -> 补材料
  - 查看详情 -> 推进状态
  - 收到提醒 -> 完成关键动作

- `用户链路示意图`
  展示从创建申请池、识别紧急事项、处理单个岗位、推进申请阶段到获得结果沉淀的完整路径。

- `阶段漏斗`
  用假设平台数据展示核心链路的分层转化。

- `真实埋点事件流`
  记录用户操作带来的本地事件变化，用于演示后续的数据分析能力。

- `分析结论 / 优化建议`
  基于现有指标给出更像真实产品后台的分析判断，而不只是停留在展示数据。

## 为什么这样设计

这个题目要求优先呈现产品结构和功能设计，不必过度追求复杂技术实现。

因此我在实现上采取了以下策略：

- 前端以单页交互 Demo 为主，快速形成可访问作品
- 用浏览器本地 `IndexedDB` 做持久化，避免引入重后端
- 通过假设平台数据和真实本地事件结合，形成“前台管理 + 后台分析”的闭环
- 在用户端强化“下一步动作”，在管理端强化“业务判断”

## 示例用户信息

当前作品中嵌入了一位示例候选人的部分真实简历信息，用于增强产品代入感。

示例字段包含：

- 姓名与基础信息
- 教育背景摘要
- 求职方向
- 代表性实习亮点
- 技能概览

这里采用的是“适度嵌入”的方式，不是把整份简历全文塞进页面，而是让产品更像一个真实候选人在使用的系统。

## 技术实现

本项目为纯静态前端页面，主要文件如下：

- [index.html](./index.html)
- [styles.css](./styles.css)
- [app.js](./app.js)
- [vercel.json](./vercel.json)

技术特点：

- 无框架依赖，便于快速部署
- 使用 `IndexedDB` 存储本地用户、申请、材料、提醒和事件数据
- 支持在浏览器中模拟新增申请、变更状态、事件沉淀等交互

## 已接入的事件示例

当前版本中，用户关键操作会写入本地事件流，例如：

- `view_dashboard`
- `view_page`
- `open_application`
- `change_stage`
- `create_application`
- `jump_from_dashboard`

这些事件会在管理端的事件流中展示，用于模拟后续真实埋点分析。

## 本地运行

### 方式一：直接打开

直接打开 [index.html](./index.html) 即可预览。

### 方式二：本地启动静态服务

在项目目录运行：

```powershell
python -m http.server 8090
```

或：

```powershell
py -m http.server 8090
```

然后访问：

```text
http://localhost:8090
```

## 部署方式

推荐直接部署到 `Vercel`。

### 推荐路径

1. 将项目上传到 GitHub 仓库
2. 在 Vercel 中导入该仓库
3. 直接点击 Deploy
4. 获取线上访问链接用于提交

当前项目已包含 [vercel.json](./vercel.json)，适合静态部署。

## 演示建议

建议按以下顺序进行 Demo：

1. 从 `用户端` 进入，查看 `今日总览`
2. 进入 `申请看板`，展示如何管理不同岗位阶段
3. 打开一个岗位详情，说明单岗位信息管理方式
4. 切到 `个人中心`，展示 AI 工作台、材料库和缺失材料分析
5. 新增一个岗位申请，或修改某个岗位状态
6. 切到 `管理端`，展示平台级指标、链路示意图、漏斗和事件流
7. 最后说明“分析结论 / 优化建议”，体现产品判断能力
